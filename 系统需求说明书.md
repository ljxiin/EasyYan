我要开发一个「考研刷题系统」Web 应用，请按规范工程化开发。

技术栈：
- 前端：Vue 3 + TypeScript + Vite
- UI：Element Plus
- 图表：ECharts
- PDF 预览：PDF.js
- 公式渲染：KaTeX
- 状态管理：Pinia
- 路由：Vue Router
- 请求：Axios
- 后端暂时先用 Mock 数据，后续接 FastAPI

系统风格：
- 后台管理系统风格
- 左侧固定菜单栏
- 顶部操作栏
- 主内容区卡片式布局
- 颜色以白色、浅灰、蓝色为主
- 圆角卡片、轻阴影、简洁高保真
- 页面比例适配 1440px / 1920px 桌面端

核心菜单：
1. 首页
2. 教材库
3. 习题库
4. 试卷库

8. 统计分析
9. 知识网络

重点页面要求：

一、首页
- 顶部统计卡片：刷题总量、正确数量、正确率、错题数量、掌握考点
- 今日任务
- 薄弱考点 TOP5，支持科目下拉切换
- 学习日历
- 最近刷题记录
- 知识掌握度分布

二、教材库
- 教材列表
- 上传教材四步流程：
  1. 上传文件
  2. 填写信息
  3. 章节识别
  4. 确认上传
- 教材字段：
  教材名称、科目、版本、类型、出版年份、作者、出版社、简介、适用阶段、标签
- 支持章节解析、整本书 OCR 解析、单节点 OCR 解析
- 笔记页面参考 Markdown 文档阅读器布局：左侧笔记列表，中间 Markdown 内容，右侧目录

三、习题库
- 题目列表
- 导入题目四步流程：
  1. 选择导入方式
  2. 预览与解析
  3. 分类与设置
  4. 确认导入
- 题目支持：题干、选项、答案、解析、考点标签
- 解析预览页支持全选确认、编辑错误题目
- 下拉展开显示答案和解析
- 考点标签放在答案解析后方，支持新增、删除

四、试卷库
- 试卷列表
- 导入试卷四步流程：
  1. 选择导入方式
  2. 预览与解析
  3. 分类与设置
  4. 确认导入
- 预览与解析页展示：
  试卷结构、题目预览、识别结果、题目编辑按钮、答案解析展开、考点标签新增删除
- 分类设置页需要标注每道题的考点标签

五、刷题
- 顶部支持：
  刷题科目下拉框
  刷题方式下拉框：按章节刷题、按考点刷题、按试卷刷题
- 选择按试卷刷题时，展示试卷列表
- 选择按章节刷题时，按科目展示章节目录
- 点击开始刷题进入刷题详情页
- 刷题详情页包含：
  题干、选项、答案解析、考点标签、收藏、笔记、纠错、上一题、下一题
- 右侧题目导航拉长展示，不需要答题统计模块

六、统计分析
包含 tabs：
- 整体概览
- 能力分析
- 题型分析
- 知识点分析
- 错题分析
- 刷题记录

每个 tab 使用 ECharts 实现图表，先用 mock 数据。

七、知识网络
- 使用图谱方式展示知识点
- 节点颜色表示掌握程度
- 支持网络视图 / 树状视图
- 右侧展示薄弱知识点 TOP10 和学习建议

八、学习计划
- 计划列表页
- 新建计划页
- 字段包括：
  计划名称、计划周期、每日学习时长、计划类型、计划目标、学习内容设置、提醒设置

九、笔记管理
- 左侧分类和标签
- 中间笔记列表
- 右侧笔记详情
- 笔记内容支持 Markdown + LaTeX

十、设置
- 账号信息
- 学习设置
- 通知设置
- 显示设置
- 数据管理
- 关于我们

开发要求：
- 所有页面先完成静态高保真 UI
- 使用 mock 数据
- 组件要拆分清晰
- 样式统一放在 global.scss 和 variables.scss
- 所有页面使用中文
- 公式使用 KaTeX 渲染
- 图表使用 ECharts
- 表格使用 Element Plus Table
- 表单使用 Element Plus Form
- 下拉框、分页、Tabs、Dialog 都使用 Element Plus
- 先不要接 OCR、AI、后端接口
- 代码要能直接运行

请先完成项目初始化、基础布局、路由、侧边栏、顶部栏和首页，然后再继续其他页面。
